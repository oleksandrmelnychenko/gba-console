# Бекенд: що фіксанути (QA-прогін 2026-07-21, dev)

Спостереження з живого прогону консолі проти `gba-api-dev` (сесія Manual QA: Товари + Облік/Платежі).

## 1. `GET /products/get` падає через нездоровʼя джерела цін (High)

- `GET /api/v1/uk/products/get?netId=...` відповідає **503** з текстом
  «Source pricing data is unavailable or unhealthy for the requested product and agreement.»
- Картка товару не має валитись через недоступність source-pricing: базові дані товару і
  ціни з джерел — різні речі, для джерел уже є окремий `GET /products/pricings/sources`
  (він при цьому відповідає 200 і навіть віддає ціни AMG/Fenix).
- Очікування: `products/get` повертає товар завжди; нездоровʼя джерела цін — максимум
  прапорець/warning у відповіді, не 503.

## 2. Час відповіді на dev (High, якщо це не інфра-ліміт)

Заміряно у Network під час прогону:

| Ендпоінт | Спостережено |
| --- | --- |
| `GET /products/get?netId=...` | 15–20+ с (pending), інколи завершується 503 з п.1 |
| `GET /products/search/advanced` | кілька секунд на перші результати |
| `GET /usermanagement/token/session` (+ `profiles/get`) | секунди на холодному старті — весь застосунок чекає цих двох запитів при кожному повному завантаженні |

`token/session`/`profiles/get` — на критичному шляху рендера (до них консоль показує
лише сплеш «Завантаження») — їм особливо потрібен швидкий шлях.

## 3. Назви типів цін зі змішаними розкладками (Medium)

- У source-pricing реальні назви містять латинські гомогліфи: «ЦP» — **латинська P (U+0050)**
  замість кириличної «Р», при кириличному «(НДС)».
- Фронт сьогодні захистився нормалізацією гомогліфів (сортування пар ЦР/ЦО1/ЦО2), але
  правильно — щоб бекенд/джерело віддавали консистентну кирилицю.

## 4. Брудні довідники на dev (Low, дані а не код)

- Статті руху грошових коштів: сміттєві записи «234234», «тест»; частина назв російською
  («Оплата покупателя», «Оплата поставщику») при україномовній консолі.
- Банки: ЄДРПОУ з хвостовою комою («14360570,»), записи без МФО/ЄДРПОУ
  (КБ «Приватбанк» м. Хмельницький).
- Якщо dev-база використовується для приймання фіч — варто підчистити, інакше QA плутає
  дані з багами.
