# UI-патерни GBA Console (еталон: Постачальники)

Зафіксовані дизайн-патерни, вироблені на екрані «Постачальники» (`/suppliers`) та
картці постачальника. **Це джерело правди для перенесення на всі інші екрани.**
Еталонні файли: `src/features/clients/pages/SuppliersPage.tsx`,
`suppliers-page.css`, `ClientEditPage.tsx`, `client-edit-page.css`,
`components/pricing/*`, спільні класи — `src/index.css`.

---

## 1. Токени і базові правила

| Що | Значення |
|---|---|
| Моно-шрифт | `font-family: var(--font-mono)` = JetBrains Mono (підключений з кирилицею через `@fontsource`, ваги 400–700) + `letter-spacing: 0` |
| Бренд-акцент | `var(--brand-orange)` (#e8782e), тінти через `rgba(var(--brand-orange-rgb), α)` |
| Фокус інпутів | **оранжевий**, не фіолетовий: глобально `border-color: var(--app-focus-border)` (index.css) |
| Фіолетовий | у навігації/акцентах **не використовуємо** (лишився тільки системний, напр. лоадери) |
| Тіні | **не використовуємо** на секціях/списках/навігації. Розділення — бордером `1px solid var(--mantine-color-gray-2)` |
| Градієнти | не використовуємо на нових екранах |
| X (закриття модалок/дроверів) | без фіолетового фокус-рінга — глобально в index.css |

## 2. Заголовки секцій — «сірий моно + оранжева крапка»

- Спільний клас: **`.app-section-title`** (index.css) — `gray-7`, моно, `::before` крапка 7px бренд-оранж.
- На edit-сторінці клієнта — близнюк `.client-section-title` (client-edit-page.css).
- Застосування: `<Text className="app-section-title" fw={600}>Заголовок</Text>`.

## 3. Блоки-секції (форми, модалки)

- `<Card className="app-section-card" withBorder radius="md" padding="md">`
- Стиль: бордер `1px gray-2`, `box-shadow: none`, компактний падінг `14px 16px`.
- Зовнішні контейнери-обгортки («шели») — **без** бордера й падінга (клас
  `client-edit-shell-card` на edit-сторінці); бордер несуть лише малі блоки.

## 4. Пілси (ролі, статуси, валюта) — `.app-role-pill`

Контурний бейдж: `колір-0` фон, `колір-2/3` бордер, `колір-8` текст, **моно 600**,
`border-radius: 7px`. Вішається на Mantine `<Badge variant="light">`.

| Варіант | Клас | Використання |
|---|---|---|
| Синій (дефолт) | `app-role-pill` | ролі («Постачальники товару», «Аналітик з закупок») |
| Сірий | `app-role-pill is-gray` | нейтральне («Резидент», валюта «Гривня») |
| Оранжевий | `app-role-pill is-orange` | увага («Нерезидент») |
| Зелений | `app-role-pill is-green` | активність («Активний»), успіх («Оплачено») |
| Червоний | `app-role-pill is-red` | проблема («Не оплачено») — коли треба, щоб кидалось в очі |

## 5. Таблиці (спільний `<DataTable>`)

**Таблиця виглядає ІДЕНТИЧНО на кожному екрані.** Всі стилі таблиці (хедер,
клітинки, зебра, hover, іконки, кебаб) — це **дефолти в
`shared/ui/data-table/data-table.css`**. Сторінки НЕ переозначають
`.data-table-*` (пер-сторінкові скіни заборонені — саме вони колись розвели
екрани). Пер-сторінковим лишається тільки структурне: висота/флекс контейнера
таблиці та слот тулбара у фільтр-барі.

Дефолти (вже в shared):

- **Хедер:** 13px / 600 / `gray-7` / **без uppercase**; нижній бордер `1px gray-2`.
- **Клітинки:** 13px / 500 / `gray-8`; нижній бордер `1px gray-2` — лінії хедера
  й рядків **однакові**.
- **Зебра:** парні рядки `gray-0`. Без transition на клітинках, без
  `translateX`/тіней на hover рядка (продуктивність!).
- **Пусті значення** — порожня клітинка, **не** тире.
- **Кебаб колонки (⋮)** — завжди видимий: `.data-table-column-menu { display: inline-flex }`.
- **Тулбар таблиці** (меню колонок ▦ + щільність ☰) — портується у фільтр-бар
  через проп `toolbarPortalTarget` (слот-div у барі). Деталі бару — §5.3.
- **Дефолтні колонки** — мінімальний робочий набір; решта прихована через
  `defaultLayout.columnVisibility` (юзер вмикає через ▦). Композитні дані —
  другим рядком у клітинці (напр. код під назвою постачальника).
- Змінюєш набір колонок → **бампни `layoutVersion`** (скидає збережені розкладки).
- Клітинки: **нативний `title`** для підказок, НЕ Mantine `<Tooltip>` (сотні
  тултіпів = лаги). Важкі функції в accessor/cell — кешувати (WeakMap по рядку).
- **Сорт-іконка** — завжди видима на сортованих колонках (прозорість 0.45 до
  активного сортування; спільний дефолт).

### 5.1 Дані в клітинках (еталон — Замовлення на Україну)

- **Грошові значення:** моно 600; **> 0 — зелений** (`green-8`, валюта-мета `green-7`), **< 0 — червоний** (`red-8`/`red-7`). Спільні класи: `.app-money` (+`.is-negative`) і `.app-money-meta` в index.css. Еталон — баланс у /accounting/supplier-organizations.

Мова даних — «Onest для назв, МОНО для чисел», сірі рівні без блідого `gray-5`:

- **Назви/сутності** (постачальник, організація): Onest **600 `gray-8`** — без
  700/near-black «чорноти».
- **Числа, коди, дати, суми** — **моно 600 `gray-8`** (сума 13px; валюта під
  сумою — моно 10px `gray-6` uppercase).
- **Мета-рядок** (№ · ств. · від · інв.): лейбли Onest `gray-6`, значення
  **моно `gray-7` 600** (ключове значення, як номер — `gray-8` 650). Патерн
  розмітки: `<span class="…meta-value [is-strong]"><span>лейбл</span><strong>значення</strong></span>`.
- **Дворядкова клітинка:** primary 12.5/600 `gray-8`, secondary 11/500 `gray-6`.
- **Без шумних чипів** (`#`, `%` тощо) біля значень — колонку вже називає хедер;
  значення показуємо голим моно-числом.
- **Вторинні сутності** в мета-рядку (договір тощо) — сірий пілс
  `app-role-pill is-gray` (на голому span додати падінги/размер).

### 5.2 Розкривні рядки (expand)

- **Шеврон:** `gray-6` у спокої → **оранжевий** на hover (без заливки) і поки
  рядок розкритий. Без вертикального роздільника після expand-колонки.
- **Вкладений блок** — бордерна картка (`1px gray-2`, радіус 8) з хедером
  вкладеної таблиці на `gray-0`; клітинки — ті самі спільні стилі.
- **Шапка вкладеної сутності** (інвойс): тип-лейбл — оранжевий моно 600;
  номер — **чорний (`gray-9`) моно 600**; іконка — **контурне коло 1px без
  фону**, коло і гліф одного кольору (`gray-7`); мета — той самий стиль
  meta-value, що в батьківському рядку.

### 5.3 Фільтр-бар (тулбар реєстру)

**Один рядок. Завжди.** Склад зліва направо:
`[фільтри: пошук / дати / комбіки] → [дії: скинути ↺, Excel/завантажити, паджинатор] → [слот ▦ ☰] → [кнопки створення]`

- **Контейнер:** `.app-filter-bar` + сторінковий клас; фон `#f8f9fa`;
  нижня межа `gray-2`; `align-items: end`; gap 10px.
- **Падінг бару — `12px 8px 14px` — живе у СПІЛЬНОМУ `filter-bar.css`**
  (`.app-filter-bar`); сторінкам його не переозначати. Горизонталь = 8px,
  щоб лейбли фільтрів стояли на одній вертикалі з текстом першої колонки
  таблиці (падінг th/td теж 8px).
- **Відступи сторінки реєстру:** `padding: 0 4px` (боки 4px, зверху — впритул,
  без «глибокої» посадки) + `Stack gap={6}`. Для console-сторінок це вже
  дефолт `.console-table-page` — не переозначати.
  Прості бари — flex (фільтр-роу `flex: 1 1 auto`); насичені — grid з
  колонкою на кожен елемент (кнопки створення = остання `auto`-колонка,
  інакше завернуться на другий рядок).
- **Лейбли над полями:** 12px, **600, МОНО, `gray-9` (майже чорні)** — НЕ
  gray-6/7. Спільне правило вже в `filter-bar.css`; сторінкам колір не
  переозначати. Фіксована висота лейбла (17px) + gap 4px — щоб поля різних
  типів вирівнювались по низу.
- **Контроли:** висота 36px, радіус 8px, бордер `gray-2`; вибрані значення
  комбіків і плейсхолдер пошуку — **моно** (12.5px/400).
- **Дії:** іконки 34px (`variant="light"`/`default"`), паджинатор
  (`<Paginator>`, лейбл моно), потім слот `toolbarPortalTarget` для ▦ ☰.
- **Кнопки створення:** `CREATE_ACTION_COLOR`, filled — головна,
  `variant="outline"` — другорядна; напис **моно 600**; сидять у самому барі
  (НЕ в хедері сторінки / PageHeaderActions).
- Одиночні `<Tooltip>` на іконках дій — дозволені (це не пер-клітинкові).

## 6. Списки з вибором (аналітики, договори)

- Рядки: білий фон, `border: 1px solid transparent` (не 0 — щоб селект не зсував
  макет), `border-radius: 8–10px`, без тіней.
- Hover: бордер `gray-3`, `translateX(3px)`.
- **Вибраний:** бордер `var(--brand-orange)`, фон **білий** (без тінтів/глову),
  **крапка 6px** бренд-оранж у правому верхньому куті (`::before`, top/right 8px);
  аватар (якщо є) — оранжеве світле коло + оранжева іконка.
- Аватар людини: `<Avatar color="gray" radius="xl" variant="light"><IconUser/></Avatar>`.

## 6.1 Степ-ліст / вертикальна навігація (еталон — картка Виробника)

Плоский текстовий ліст без карток — для степ-навігації в шитах і будь-яких
лівих нав-лістів (референс: `client-edit-page.css`, `.client-edit-nav-*`):

- **Шапка над лістом** (для сутності): назва **моно** + зелений/сірий Switch
  статусу; під нею — роль у пілсі `app-role-pill`.
- **Пункт:** моно 13px / 500 / `gray-7`; без фону, бордера й тіней;
  `min-height: 40px`, радіус 8, `padding-left: 18px` (місце під крапку),
  `padding-right: 6px`; ніяких номерів/іконок зліва.
- **Активний:** текст **оранжевий 600** + **оранжева крапка 7px зліва**
  (`::before`, left 8px, по центру вертикалі — та сама крапка, що в заголовків
  секцій: пункт «веде» до свого блоку) + **шеврон одразу після тексту**
  (оранжевий, видимий постійно; label не розтягувати — шеврон тулиться до
  тексту, не до краю).
- **Hover:** текст `gray-9` + `translateX(3px)`; шеврон з'являється сірим.
- Прозорий фон і на hover (`background: transparent !important` проти
  Mantine subtle).

## 6.2 Рейка-ліст біля гріда (двопанельний реєстр)

Еталони — `/users` (ролі + користувачі) і `/transporters` (типи + перевізники).
Лівий плоский ліст-фільтр поруч із таблицею в рамці, всередині спільного шела
(під фільтр-баром):

- **Workspace:** `gap: 12px`, падінг `10px 8px 8px`; ліва колонка фіксована
  (~232–280px), права — `minmax(0, 1fr)` / `flex: 1`.
- **Рейка:** НА БІЛОМУ — без фонової заливки колонки і без вертикального
  бордера-роздільника (розділення дає рамка гріда). Flex-колонка; скрол
  списку — `flex: 1 1 auto; min-height: 0; overflow: auto` (ніяких
  `calc(100vh - N)`).
- **Заголовок рейки** — `app-section-title` (моно + оранжева крапка), 13/600,
  `padding-left: 8px`.
- **Пункти** — за §6.1: плоскі моно 13/500 `gray-7`, активний — оранжевий 600 +
  крапка зліва; **лічильник** праворуч — маленьке моно `gray-6` (в активного —
  оранжеве).
- **Грід у рамці:** `border: 1px solid gray-2; border-radius: 10px;
  overflow: hidden` — DataTable заповнює рамку (`height="100%"`, скрол
  всередині).
- Панелі тягнуться на **однакову висоту** (flex stretch); скролиться контент
  панелей, не сторінка.

## 7. Модалки

- Тайтл — **моно**; для сутностей зі статусом — зелена/сіра крапка-статус перед назвою.
- Ширини: компактні дії ~496px; паспорт — `lg`; великі форми — широка модалка

### 7.1 Попап дій рядка — ОБОВ'ЯЗКОВИЙ патерн (еталон — Виробники)

Клік по рядку реєстру відкриває саме такий попап. Один вигляд для всіх екранів
(еталони: `SupplierDetailDrawer` у SuppliersPage, `ClientActionsModal` у ClientsPage):

- `<AppModal centered size={496}>`; **тайтл** — моно + крапка-статус (9px,
  зелена з глоу коли активний / сіра).
- **Список дій** — контейнер `Stack className="app-modal-actions"`; кожна дія:
  `Button fullWidth justify="flex-start" variant="subtle" color="dark" size="md"`
  з іконкою в `<span className="app-action-icon">` (Lucide, size 20,
  `color: var(--mantine-color-gray-7)`). Спільні класи в `index.css` дають:
  контурне коло 36px без фону; hover — коло+іконка+текст оранжеві, фон
  прозорий.
- Після списку — `<Divider />`, під ним **тогл активності**
  (`variant="light"`, gray/green залежно від стану).
- Жодних інфо-блоків з даними в цьому попапі — тільки дії (дані живуть у
  картці/паспорті).

## 8а. Кнопки з «мертвого» хедера сторінки — переносити в бар

Хлібні крихти / верхній тулбар сторінки вимкнені (`ConsoleContentHeader`
закоментований), але ~20 екранів досі рендерять кнопки створення через
`<PageHeaderActions>` у верхній слот. **При порті кожного екрана цю кнопку
обов'язково переносити у фільтр-бар** (див. §5.3) і прибирати
`PageHeaderActions` з файла. Приклади перенесення: SuppliersPage, ClientsPage.

### 7.2 Деталка сутності в шиті (еталон — Підзвітна витрата)

Right-drawer з даними одного запису (`/accounting/consumable-services`, деталка):

- **Тайтл дровера** — моно. **Шапка (hero):** назва сутності — **моно 17/500**
  (не жирна, без декоративної іконки), під нею сірий мета-рядок; поруч —
  **пілси статусів** різними кольорами (зелений/оранжевий/**червоний**/сірий —
  статус має кидатись в очі); невідомий статус — пілс **ховаємо** (не «—»).
- **Метрики** (сума/оплачено/кількість) — патерн підсумків: без коробок і
  кольорових смуг; сірий моно-лейбл з **оранжевою крапкою** + велике
  **моно**-значення (18/600), дрібна мета під ним.
- **Секції** — заголовок `app-section-title` (моно + крапка), **без
  лінії-роздільника** під заголовком і без іконок.
- **Поля** — **leader-рядки** «лейбл ——— значення»: лейбл `gray-8` 500 зліва,
  тонка лінія `gray-3` (flex-заповнювач, `translateY(-3px)` до базлайну),
  значення **600 `gray-9`** праворуч (text-align right). Сітка пар — 2 колонки,
  `row-gap: 12px; column-gap: 28px`. Пусте значення — просто порожній рядок.
- Реалізація leader-рядка: контейнер flex + `::before` як лінія з `order: 2`
  (label `order: 1`, value `order: 3`).

## 8. Кнопки

## 9. Форми (edit-сторінка)

- Хост — `AppDrawer size="standard"` (900px), gap між секціями `md`.
- Степ-навігація зліва: плоскі текстові пункти (моно 13, без фону/бордера),
  активний — **оранжевий текст + оранжева крапка зліва**, шеврон одразу після
  тексту; hover — темніший текст + `translateX(3px)`.
- Комбіки — стиль тулбара: висота 36px, радіус 8px, бордер `gray-2`, **моно 13px**.
- Чекбокси — **оранжеві** (`:checked` → бренд-оранж). Свічі статусу — зелені.
- Сітки полів: `SimpleGrid` з `style={{ alignItems: 'end' }}` (щоб довгі лейбли
  не зсували поля), групи з кнопками — `wrap="nowrap"`.
- Лейбли полів — звичайний Onest (НЕ моно). Моно — тільки заголовки секцій,
  кнопки, комбіки.
