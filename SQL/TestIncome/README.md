## Відомі особливості джерела

- У MAYER клітинки `C5:C27` мають зовнішні `VLOOKUP`, але в контрольному файлі збережені їхні кешовані значення. Не перезберігати цей файл в Excel без заміни формул на значення.
- Повторюються артикули MAYER `13460CNT`, `101-U11-108-2 MAYER` та NOIR `1231528712 NR`, але їхні кількість/ціна різні. Імпортер не повинен зливати ці рядки.
- У PDF AYMEKS текстовий шар містить інше написання номера рахунку; звіряти з папкою `AYM2026000000665`.
- У PDF OSMANLI зазначено `б/н 17.06.26`; контрольна папка має `OSD2026000000030 19.06.2026`.

## Критерії повної звірки системи

- Кількість документів, товарних рядків, коди та Qty — точна рівність.
- Грошові поля — точність до `0,01` на кожному рядку, а не лише збіг загального підсумку.
- Нормалізований SHA-256 кожного набору рядків фіксує порядок, код, УКТЗЕД, ціну, Qty, митну вартість, мито та ПДВ; перестановка або компенсація сум між рядками ламає тест.
- Контрольна дата МД для всіх семи декларацій — `20.07.2026`.
- Три повторні артикули мають залишитися окремими рядками.
- Фізичний рядок визначається власним `SupplyInvoiceOrderItemID`, а не комбінацією товару, Qty і ціни; навіть повністю однакові за цими полями рядки не можна зливати.
- Після повторного завантаження не повинні з'являтися нові специфікації, приходи, залишки або рухи.
- Після розміщення сума Qty у приході та доступному залишку має дорівнювати Qty паклиста.
- Для REMI та FSS результат додатково звіряється рядок у рядок з уже проведеним 1С-еталоном.
- Перевірка виконується лише на ізольованій тестовій копії БД; цей набір файлів не є дозволом змінювати DEV або PROD.

## Автоматичний gate

Fixture-контракт джерел запускається з консолі:

```bash
rtk npm test -- tests/testIncomeFixtures.test.ts
```

Повний backend E2E навмисно дозволено запускати лише на disposable-копії
`ConcordDb_V5_Pricing82Test`. Спочатку копія перевіряється, очищається та мігрується:

```bash
rtk /root/projects/gba-server/scripts/reset-testincome-disposable-clone.sh
```

Потім усі TestIncome SQL-сценарії запускаються однією командою без виведення секрету:

```bash
rtk bash -lc 'base="$(< /root/projects/gba-infra/secrets/dev/ConnectionStrings__RemoteConnectionString)"; base="Server=127.0.0.1,1433;${base#*;}"; clone="${base/Database=ConcordDb_V5/Database=ConcordDb_V5_Pricing82Test}"; TEST_INCOME_ROOT=/root/projects/gba_console/SQL/TestIncome PRICING82_INTEGRATION_CONNECTION_STRING="$clone" PRICING82_INTEGRATION_REQUIRED=1 rtk dotnet test /root/projects/gba-server/tests/Global.Business.Assistant.Platform.Actors.Tests/Global.Business.Assistant.Platform.Actors.Tests.csproj --no-build --no-restore --filter FullyQualifiedName~TestIncome'
```

Контрольний результат від 23.07.2026: `15/15` TestIncome SQL-тестів та `11/11`
fixture-тестів пройшли. E2E створив рівно 7 приходів, 507 income/consignment
items, 507 рухів і розміщень, 504 товарні залишки із сумарним Qty `21 382`,
після чого підтвердив незмінність усіх цих даних при replay та повністю очистив
disposable-копію. Окремі fault-сценарії підтвердили повний rollback приходу та
консигнації, атомарність повторного імпорту рахунка/специфікації, заборону
видалення фізичного рядка після входження в паклист, strict retry без дублів,
transactional outbox та збереження exact source total під час legacy GET/update
round-trip.

Цей gate починається з уже сформованих у тестовій БД рахунків і паклистів,
рядки яких точно відтворені з контрольних Excel. Він використовує реальний Excel
parser, supply/income/consignment actors та SQL repositories, але не замінює
окремий browser/API smoke створення початкового рахунку й паклиста.

Source-boundary HP закрито без зміни контракту чотиризначної ціни за одиницю.
П'ять рядків і далі неможливо точно відновити лише як `UnitPrice × Qty`, тому
сервер зберігає окремий `SourceTotalNetPrice`. Для повного рядка використовується
ця сума, для частини паклиста — пропорція `SourceTotalNetPrice / invoice Qty ×
child Qty`, а за відсутності source total лишається старий fallback. У результаті
збережений HP total та `SupplyInvoice.NetPrice` дорівнюють source `14 841,69`;
митна вартість, мито, ПДВ, доставка й downstream-собівартість також проходять
рядкову звірку.
